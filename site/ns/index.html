License Database vocabulary

The License Database vocabulary extends The Creative Commons Rights Expression Language (CC REL) with additional properties to describe existing copyright and database licenses in more detail.

The License Database project uses this vocabulary to describe existing free software, open source software and free culture licenses used today.

Classes

li:License

A set of requests/permissions to users of a Work, e.g. a copyright license, database rights license, trademark license, patent grant, etc..

li:LicenseException

A set of permissions granted in addition to the permissions granted by a copyright license.

License Properties

li:id

The identifier assigned to the license by the LicenseDB project.

li:hasPlainText

A license is available as a plain text formatted resource. This is usually the preferred format to include in packaged software project.

li:compatible

A License is compatible with a License.

li:hasEarlierVersion

A License is an earlier version of License.

li:hasLaterVersion

A License is a is a later version of License. This property can be used to find applicable licenses for a work which is licensed under a versioned license with an "or any later version" clause.

li:orLaterNotices

The license notices typically used in project documentation or source files to put the work under this license (with an "or any later version" clause).

li:orLaterShortNotice

The license notice typically used in the user interface of a work to put the work under this license (with an "or any later version" clause).

li:versionOnlyNotices

The license notices typically used in project documentation or source files to put the work under this license (with an "only this version" clause if the license is versioned).

li:versionOnlyShortNotice

The license notice typically used in the user interface of a work to put the work under this license (with an "only this version" clause if the license is versioned).
